# 📜 Night Gym Academy - Project Rules

Version: 1.0

Project Name:
Night Gym Academy

Parent Company:
YFLB

Founder:
Youssef El Basuony

---

# 1. Project Philosophy

Night Gym Academy is not just a website.

It is a professional educational platform designed to become one of the world's leading fitness education ecosystems.

Every feature must be designed with scalability, maintainability, and long-term growth in mind.

---

# 2. Development Order

Every feature must follow this order:

1. Planning
2. HTML
3. CSS
4. JavaScript
5. Testing
6. Responsive Design
7. Optimization

Never skip a step.

---

# 3. Phase Rules

Never move to a new Phase until the current Phase is completed.

A Phase is considered complete only after:

✔ HTML finished

✔ CSS finished

✔ JavaScript finished

✔ Responsive completed

✔ Testing completed

---

# 4. Coding Rules

Write clean code.

Write reusable code.

Do not duplicate code.

Do not write unused code.

Keep files organized.

Use meaningful names.

Comment complex code when necessary.

---

# 5. UI Rules

Keep the design consistent.

Follow the Night Gym design system.

Use the same colors.

Use the same spacing.

Use the same typography.

Use reusable components whenever possible.

---

# 6. Content Rules

All educational content must be scientifically accurate.

Avoid misinformation.

Use trusted references whenever possible.

Organize information from beginner to advanced.

Explain concepts clearly.

---

# 7. Exercise Rules

Every exercise should include:

Arabic Name

English Name

Scientific Name

Target Muscle

Secondary Muscles

Equipment

Execution

Breathing

Tempo

Common Mistakes

Safety Tips

Variations

Alternatives

---

# 8. Lesson Rules

Every lesson should contain:

Objectives

Explanation

Images

Examples

Important Notes

Summary

Quiz

References

---

# 9. Question Rules

Every question should include:

Difficulty

Correct Answer

Explanation

Reference

Tags

Category

---

# 10. Future Proof

Every feature should be expandable.

Never build something that prevents future improvements.

Think long-term.

---

# 11. Documentation

Major changes should be documented.

Project files should always remain organized.

---

# 12. General Rule

Quality is more important than speed.

Build once.

Build correctly.

Improve continuously.
