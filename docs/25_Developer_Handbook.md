# Development Philosophy

Always build:

- Clean Code
- Reusable Components
- Modular Architecture
- Scalable Features
- Responsive Design
- Maintainable Code

Never build quick fixes that make future development harder.

---

# UI Philosophy

The UI must always remain:

- Premium
- Modern
- Minimal
- Dark Theme
- Easy to Navigate
- Consistent

Never redesign existing pages without approval.

---

# Coding Rules

HTML

- Use semantic HTML.
- Keep structure clean.
- Avoid unnecessary nesting.

CSS

- Reuse existing classes.
- Avoid duplicated styles.
- Keep sections organized.
- Follow the Design System.

JavaScript

- Use meaningful variable names.
- Write reusable functions.
- Avoid global variables.
- Keep code modular.

---

# File Organization

Each file should have one clear responsibility.

Avoid placing unrelated code in the same file.

Every important feature should be documented.

# Accessibility

Use accessible HTML.

# Security

Never expose sensitive information.

Validate user input.

Sanitize user input.

Protect user data.

Follow Security.md.

---

# Git Rules (Future)

Write meaningful commit messages.

Commit small logical changes.

Do not commit broken code.

---

# Workflow

Every task follows:

1. Read Documentation

2. Plan

3. Build

4. Test

5. Review

6. Update Documentation

7. Complete

---

# Bug Fixing

Never hide bugs.

Fix the root cause.

Test before closing.

Document major fixes.

---

# Code Reviews

Every new feature should be reviewed before being considered complete.

---

# Design Consistency

Always use:

Existing colors

Existing spacing

Existing typography

Existing components

Do not invent new styles without approval.

---

# AI Usage

AI tools may assist development.

However:

- Verify generated code.
- Follow project standards.
- Never blindly copy code.
- Keep documentation updated.

---

# Long-Term Vision

Write today's code as if it will still be used five years from now.

Prioritize readability, maintainability, and scalability.

---

# Final Principles

Quality before speed.

Consistency before creativity.

Documentation before implementation.

User experience before visual effects.

Scientific accuracy before popularity.

Every improvement should make Night Gym Academy a better educational platform.
